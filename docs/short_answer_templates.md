# 简答题模板

## 1. 传统视觉与深度学习的对比与选型

总论点：传统视觉和深度学习不是替代关系，而是要按任务稳定性、样本条件、实时性和可解释性综合选型。

可直接改写的结构：

1. 先看任务特征。若目标规则明确、背景稳定、几何约束强，例如读码、尺寸测量、固定治具定位，传统方法往往更快更稳。
2. 再看数据与复杂度。若场景存在遮挡、反光、纹理变化大、类间差异细微，例如表面缺陷和复杂目标检测，深度学习更有优势。
3. 再看工程落地。传统视觉开发周期短、可解释性强，但泛化有限；深度学习前期数据成本高，但扩展复杂场景能力更强。
4. 最后给混合方案。工业项目里常用“传统预处理 + 深度模型 + 规则后处理”，既提升精度，也降低误检漏检。

工业化落点：选型不能只看离线精度，还要看误报成本、上线维护成本、换线复用能力和现场算力条件。

## 2. 工业缺陷检测系统构成与流程

总论点：缺陷检测系统是“成像系统 + 算法系统 + 业务闭环”的组合，任何一环薄弱都会影响最终效果。

可直接改写的结构：

1. 成像层。包括相机、镜头、光源、触发和工装定位，目标是把缺陷稳定拍清楚。
2. 数据层。完成样本采集、缺陷定义、标注规范、训练集验证集划分，保证数据代表真实工况。
3. 算法层。可采用传统分割、特征匹配或深度学习检测/分割模型，并配合阈值和规则后处理。
4. 部署层。把模型放到工控机或边缘设备，联通PLC、MES或机器人执行端，形成告警或剔除动作。
5. 迭代层。根据现场误检、漏检和新缺陷类型持续回流数据，闭环优化。

工业化落点：很多缺陷问题不是单纯换模型就能解决，往往要从光学设计、样本体系和判定规则一起调整。

## 3. 嵌入式部署优化思路与现场注意点

总论点：嵌入式部署的核心不是把训练模型原样搬过去，而是在精度、时延、功耗和稳定性之间做平衡。

可直接改写的结构：

1. 模型侧优化。选择轻量网络，并结合量化、剪枝、蒸馏、算子融合减少计算量。
2. 推理侧优化。使用 ONNX Runtime、TensorRT 或硬件专用加速库，提高吞吐与稳定性。
3. 系统侧优化。通过多线程、流水线、异步 IO、缓存和批处理降低端到端时延。
4. 现场侧注意点。关注温度、电磁干扰、网络抖动、相机曝光漂移、版本回滚和异常日志。
5. 验证侧。不能只做实验室测试，还要做现场长稳测试、边界工况测试和断电恢复测试。

工业化落点：嵌入式最常见问题不是平均速度不够，而是极端样本下时延抖动大、内存泄漏和现场环境变化。

## 4. 工业制造中 AI 视觉的趋势与挑战

总论点：AI 视觉正从单点检测走向多任务协同和全流程智能化，但落地难点仍集中在数据、鲁棒性和系统集成。

可直接改写的结构：

1. 趋势一，模型能力增强。检测、分割、OCR、三维视觉和多模态技术正在融合，系统更接近通用感知。
2. 趋势二，边缘部署普及。越来越多算法直接部署在产线边缘设备上，强调实时性和本地闭环。
3. 趋势三，数据闭环建设。企业开始重视数据资产、持续训练和在线监控，而不是一次性交付模型。
4. 挑战一，场景变化大。反光、污渍、振动、工件批次变化会持续冲击模型泛化。
5. 挑战二，样本稀缺。缺陷样本少、标注贵、异常类型长期变化，导致模型维护成本高。
6. 挑战三，集成复杂。算法还要与机械、电气、PLC、MES和安全策略协同。

工业化落点：未来竞争力不只在单个模型精度，而在“稳定成像 + 高效模型 + 数据闭环 + 工业系统集成”能力。
